# Karpathy Guidelines

Use these rules to reduce common coding-agent mistakes:

1. Think before coding. State assumptions, surface tradeoffs, and ask when the request is genuinely ambiguous.
2. Prefer the simplest implementation that fully solves the request. Do not add speculative flexibility.
3. Make surgical changes. Match existing style and touch only lines that trace to the task.
4. Execute against verifiable success criteria. Define the check, run it, and iterate until it passes.
